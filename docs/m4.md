# M4：SFT 冷启动

## 本模块回答什么问题

M4 回答的是：

> 在进入强化学习之前，模型能否先学会基本的“搜索推理格式”和工具调用习惯？

SFT 冷启动的作用不是追求最终性能，而是为 GRPO 提供一个可训练的起点。

---

## 为什么它对面试重要

这一模块能体现你对 Search Agent 训练链路的理解：

- 为什么不能直接从 base model 上 RL
- 为什么要先教会标签格式和工具调用
- 为什么 SFT 在这里是”可训练性保障”，不是最终目标
- `<information>` token 的 loss mask 是 SFT 的关键工程决策：检索结果属于环境注入，模型不应学习生成它，必须在 SFT loss 计算中屏蔽

---

## 范围

### M4 要完成的事

- 定义统一的 system prompt / 标签协议
- 构造 SFT 冷启动数据
- 训练模型学会基本的 `<think> / <search> / <information> / <answer>` 输出格式
- 产出可作为 GRPO 起点的 checkpoint

### M4 不做的事

- 不追求 SFT 本身的最终最好效果
- 不把 SFT 当成项目终点
- 不在本地做正式大模型训练

---

## 本地与远端分工

### 本地

- 用极小数据做 `--dry-run` 或 `--smoke-test`
- 验证数据读取、token 化、loss 计算、训练 step 是否能启动

### 远端

- 使用真实模型和真实 LoRA / QLoRA 配置
- 完成正式 SFT
- 产出后续 GRPO 要加载的 checkpoint

---

## 关键输出物

- SFT 冷启动数据
- SFT smoke test 可运行入口
- 正式 SFT checkpoint

---

## 通过标准

满足以下条件即可认为 M4 完成：

- 本地 smoke test 能真实跑一个最小训练 step
- 远端正式 SFT 能产出 checkpoint
- 模型能稳定生成正确的基础标签结构

---

## 面试讲法

M4 对外可以这样讲：

> 我把 SFT 作为强化学习前的可训练性保障，而不是把它当成最终方案。它的核心作用是让模型先学会 Search Agent 所需的输出结构和工具调用模式，为后续 multi-turn GRPO 提供一个稳定起点。
