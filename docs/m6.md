# M6：GRPO 主训练与课程学习

## 本模块回答什么问题

M6 回答的是：

> 在 SFT、检索服务、multi-turn 环境和 reward 设计都准备好后，1.5B 小模型是否真的能被训练出搜索增强推理能力？

这里是项目从“模块堆叠”走向“真正方法验证”的关键阶段。

---

## 为什么它对面试重要

M6 决定了你能否向面试官证明：

- 这个项目不只是一个工程骨架
- multi-turn retrieval + GRPO 这条主线确实可以训练
- 课程学习、混合检索、过程奖励是可验证的变量，而不是口号

---

## 范围

### M6 要完成的事

- 把 SFT checkpoint、retriever、rollout、reward 接到同一训练流程中
- 完成最小 GRPO baseline
- 逐步加入课程学习与其他改进项
- 形成正式 checkpoint 与训练日志

### M6 不做的事

- 不要求一次性把所有增强项全部塞进第一版训练
- 不要求本地完成正式训练

---

## 推荐推进顺序

### 第一层：本地 smoke test

- GRPO 入口脚本能启动
- rollout / reward / trainer 至少能走通一次最小闭环

### 第二层：远端 baseline

- 用正式模型跑最小可训练版本
- 验证训练是否稳定

### 第三层：方法增强

默认顺序：先 baseline → 再课程学习 → 再 Hybrid Retrieval → 再过程奖励

#### 课程学习（三阶段）

| 阶段 | 数据集 | 步数 | 说明 |
|------|--------|------|------|
| Stage 1（easy） | HotpotQA easy | 300 步 | 建立基础 reward 信号 |
| Stage 2（medium） | HotpotQA medium + MuSiQue 2hop | 500 步 | 迁移到轻度多跳 |
| Stage 3（hard） | HotpotQA hard + MuSiQue 3-4hop | 700 步 | 强化多跳推理能力 |

每阶段从上一阶段 checkpoint 继续训练（`resume_from_checkpoint`）。

#### 混合检索

- 对比 BM25-only 与 BM25+Dense+RRF（Hit@3 + 最终 EM）

#### 过程奖励

- 对比 F1-only 与 F1+Format+SearchQuality（前 200 步 reward 收敛曲线）

---

## 关键输出物

- 正式 `grpo_train.py`
- 课程学习配置
- GRPO checkpoint
- 训练日志与关键曲线

---

## 通过标准

满足以下条件即可认为 M6 完成：

- 本地 smoke test 验证 GRPO 链路成立
- 远端最小 baseline 训练跑通
- 训练过程没有明显崩溃
- 至少产出一版可评测的模型 checkpoint

---

## 面试讲法

M6 对外可以这样讲：

> 我没有直接堆复杂方法，而是先建立最小可训练的 GRPO baseline，再逐步加入课程学习、混合检索和过程奖励。这样每个设计改动都能被单独验证，也更容易分析它们对训练稳定性和最终效果的影响。
