## M3：检索服务器搭建（BM25 + E5 + RRF + FastAPI）

**本模块目标**：搭建一个本地的混合检索 HTTP 服务——输入 query，返回 top-K passage。训练和评测时，Agent 会通过 HTTP 调用这个服务。

**本模块产出**：一个能在 AutoDL 上稳定运行的 FastAPI 服务，监听 `127.0.0.1:8000`，提供 `POST /retrieve` API（接口契约见 CLAUDE.md §4.1）。

**本模块耗时估计**：代码 1 天；构建 BM25 索引 **30-60 分钟**；构建 Dense 索引 **4-6 小时**（单卡 3090）；总计 **1-2 天**。

**成本预估**：AutoDL 3090 跑 Dense 索引约 **8-10 元**；BM25 用无卡模式约 **0.1 元**；总计 **~10 元**。

### 一、架构全景图

```
训练进程(GRPO rollout)
      │
      │ HTTP POST localhost:8000/retrieve
      │ {"query": "Steve Jobs wife", "topk": 3}
      ▼
┌─────────────────────────────────────┐
│      FastAPI 检索服务器              │
│                                     │
│  ┌──────────────┐  ┌──────────────┐ │
│  │ BM25 Searcher│  │ Dense Searcher│ │
│  │  (Pyserini)  │  │  (E5 + FAISS) │ │
│  └──────┬───────┘  └───────┬──────┘ │
│         │                   │        │
│         └────── RRF 融合 ────┘        │
│                  │                   │
│                  ▼                   │
│             Top-K passages           │
└─────────────────────────────────────┘
      │
      │ JSON 响应
      ▼
  训练进程得到检索结果,继续 rollout
```

**关键设计**：

- **两个独立检索器并行查**，各取 top-K（K 比最终输出大一些，比如最终要 3 条就各取 10 条）
- **RRF 融合**：基于排名倒数加权，不需要分数归一化
- **服务和训练都在 AutoDL 同一台机器**：走 `localhost` 通信，零网络延迟
- **BM25 和 Dense 索引都预先构建好保存到磁盘**：服务启动时加载到内存

```python
"""构建 BM25 索引"""
import subprocess
import json
from pathlib import Path

CORPUS_PATH = Path("data/raw/wiki-18-corpus/wiki-18.jsonl")
INDEX_DIR = Path("data/indexes/bm25")
COLLECTION_DIR = Path("data/tmp/bm25_collection")

# Pyserini 需要 corpus 是特定的 JSON Collection 格式
# 每行: {"id": "xxx", "contents": "标题\n正文"}
# 我们的 wiki-18.jsonl 已经是这个格式,直接用!


def prepare_collection():
    """Pyserini 要求 corpus 文件放在一个目录里,直接软链接即可"""
    COLLECTION_DIR.mkdir(parents=True, exist_ok=True)
    target = COLLECTION_DIR / "wiki-18.jsonl"
    if not target.exists():
        # 用软链接避免复制 26GB
        target.symlink_to(CORPUS_PATH.absolute())
    print(f"Collection 准备好: {COLLECTION_DIR}")


def build_index():
    """调用 Pyserini 命令行建索引"""
    INDEX_DIR.mkdir(parents=True, exist_ok=True)
    
    cmd = [
        "python", "-m", "pyserini.index.lucene",
        "--collection", "JsonCollection",
        "--input", str(COLLECTION_DIR),
        "--index", str(INDEX_DIR),
        "--generator", "DefaultLuceneDocumentGenerator",
        "--threads", "8",           # CPU 线程数,根据机器调整
        "--storePositions",         # 存位置信息(支持短语查询)
        "--storeDocvectors",        # 存 doc vectors
        "--storeRaw",               # 存原文(方便返回给模型)
    ]
    
    print("开始建 BM25 索引...")
    print("命令:", " ".join(cmd))
    subprocess.run(cmd, check=True)
    print(f"✅ BM25 索引完成: {INDEX_DIR}")


if __name__ == "__main__":
    prepare_collection()
    build_index()
```

### 三、Dense 索引构建（E5 + FAISS）

#### 要点

**这是 M3 最耗时、最耗钱的部分**——29M passage 每条都要过一遍 E5 模型生成 embedding。

- 模型：**intfloat/e5-base-v2**（768 维，性能和速度平衡好）
- 索引类型：**FAISS IndexFlatIP**（精确搜索，29M 规模下内存够用）
- 耗时：**单卡 3090 跑 4-6 小时**
- 索引大小：约 **90GB**（29M × 768 维 × 4 字节）

#### 一个关键决策：用 IndexFlatIP 还是 IndexIVFFlat

|索引类型|精度|查询速度|磁盘占用|构建速度|
|---|---|---|---|---|
|**IndexFlatIP**|100%|较慢（500ms/query）|~90GB|最快|
|IndexIVFFlat|~98%|快（10ms/query）|~90GB|慢（需训练聚类）|
|IndexHNSW|~99%|最快（1ms/query）|~130GB|最慢|

> 备注：CLAUDE.md §6.4 曾提到 HNSW "内存减半" 作为 fallback——这个说法只在**不用 memmap、全量读入 RAM** 的情况下成立。我们用 memmap，驻内存只有 15-20GB（详见坑点 8），HNSW 不会带来 RAM 收益，但会增加磁盘和构建时间。

**推荐 IndexFlatIP**，理由：

- 训练期间 query 频率不高（一次 rollout 最多 3 次搜索，见 CLAUDE.md §3.1），500ms/query 能接受
- 构建最快最省事
- 精度 100%，不引入近似误差污染你的 reward 信号

#### 坑点 4：90GB 索引超过默认磁盘

AutoDL **默认系统盘 50GB**，加上 Wiki 原文 26GB、BM25 索引 5GB，已经占了 31GB。再加 90GB 的 Dense 索引**绝对会爆**。

**两个解决方案**：

**方案 A（推荐）：开机时增加数据盘到 200GB** AutoDL 实例详情页可以"扩容数据盘"，1 元/月/GB。200GB × 30 天 = 6 元/月，实际用几天也就 1-2 元。

**方案 B：用 IVF 索引压缩空间** 改用 `IndexIVFPQ` 可以把 90GB 压缩到 15GB 左右，但构建更慢且精度下降 2-3%。**不推荐**，除非真的磁盘受限。

#### 代码：`retrieval/build_dense_index.py`

```python
"""用 E5-base-v2 给 Wiki 语料生成 embedding,构建 FAISS 索引"""
import os
import json
import numpy as np
import torch
import faiss
from pathlib import Path
from tqdm import tqdm
from transformers import AutoTokenizer, AutoModel
from torch.utils.data import DataLoader, Dataset

os.environ['HF_ENDPOINT'] = 'https://hf-mirror.com'

CORPUS_PATH = Path("data/raw/wiki-18-corpus/wiki-18.jsonl")
INDEX_DIR = Path("data/indexes/dense")
INDEX_DIR.mkdir(parents=True, exist_ok=True)

MODEL_NAME = "intfloat/e5-base-v2"
BATCH_SIZE = 256  # 3090 24GB 上 E5-base 的甜点值
MAX_LENGTH = 256  # passage 长度截断,Wiki 100-word split 基本不会超过 256 tokens
DIM = 768


class PassageDataset(Dataset):
    def __init__(self, jsonl_path):
        self.data = []
        with open(jsonl_path, encoding='utf-8') as f:
            for line in f:
                ex = json.loads(line)
                # E5 官方推荐 passage 前面加 "passage: " 前缀
                self.data.append({
                    "id": ex["id"],
                    "text": f"passage: {ex['contents']}"
                })
        print(f"加载 {len(self.data)} 条 passage")
    
    def __len__(self):
        return len(self.data)
    
    def __getitem__(self, idx):
        return self.data[idx]


def collate_fn(batch, tokenizer):
    texts = [item["text"] for item in batch]
    ids = [item["id"] for item in batch]
    encoded = tokenizer(
        texts, padding=True, truncation=True,
        max_length=MAX_LENGTH, return_tensors="pt"
    )
    return ids, encoded


def average_pool(last_hidden_states, attention_mask):
    """E5 用 mean pooling(论文指定)"""
    last_hidden = last_hidden_states.masked_fill(
        ~attention_mask[..., None].bool(), 0.0
    )
    return last_hidden.sum(dim=1) / attention_mask.sum(dim=1)[..., None]


@torch.no_grad()
def encode_corpus():
    device = "cuda"
    tokenizer = AutoTokenizer.from_pretrained(MODEL_NAME)
    model = AutoModel.from_pretrained(MODEL_NAME).to(device).eval()
    # 用 fp16 加速
    model = model.half()
    
    dataset = PassageDataset(CORPUS_PATH)
    loader = DataLoader(
        dataset,
        batch_size=BATCH_SIZE,
        shuffle=False,
        num_workers=4,
        collate_fn=lambda b: collate_fn(b, tokenizer),
        pin_memory=True,
    )
    
    # 预分配 numpy memmap,避免 90GB 直接占满内存
    total = len(dataset)
    embeddings = np.memmap(
        INDEX_DIR / "embeddings.npy",
        dtype='float32', mode='w+',
        shape=(total, DIM)
    )
    ids_list = []
    
    cursor = 0
    for ids_batch, encoded in tqdm(loader, total=len(loader)):
        encoded = {k: v.to(device) for k, v in encoded.items()}
        outputs = model(**encoded)
        emb = average_pool(outputs.last_hidden_state, encoded['attention_mask'])
        # L2 归一化(为了用 IndexFlatIP 做余弦相似度)
        emb = torch.nn.functional.normalize(emb, p=2, dim=1)
        
        bs = emb.size(0)
        embeddings[cursor:cursor+bs] = emb.float().cpu().numpy()
        ids_list.extend(ids_batch)
        cursor += bs
    
    embeddings.flush()
    
    # 保存 ID 映射
    with open(INDEX_DIR / "ids.txt", "w") as f:
        for id_ in ids_list:
            f.write(f"{id_}\n")
    
    print(f"✅ Embedding 完成,总 {cursor} 条")
    return cursor


def build_faiss_index(total):
    """构建 FAISS IndexFlatIP"""
    print("加载 embeddings 到 FAISS...")
    embeddings = np.memmap(
        INDEX_DIR / "embeddings.npy",
        dtype='float32', mode='r',
        shape=(total, DIM)
    )
    
    # IndexFlatIP:精确内积搜索(因已归一化,等价余弦相似度)
    index = faiss.IndexFlatIP(DIM)
    
    # 分批 add,避免一次性 copy 90GB 到 FAISS 内部
    chunk_size = 100000
    for i in tqdm(range(0, total, chunk_size)):
        chunk = embeddings[i:i+chunk_size].copy()  # memmap 转 np array
        index.add(chunk)
    
    faiss.write_index(index, str(INDEX_DIR / "faiss.index"))
    print(f"✅ FAISS 索引保存: {INDEX_DIR / 'faiss.index'}")


if __name__ == "__main__":
    total = encode_corpus()
    build_faiss_index(total)
```

### 四、RRF 融合算法

#### 要点

RRF（Reciprocal Rank Fusion）的公式非常简单：

```
                   1
RRF(d) = Σ  ─────────────────
         i   k + rank_i(d)
```

- `rank_i(d)`：文档 d 在第 i 个检索器里的排名（从 1 开始）
- `k`：平滑常数，**通常取 60**（Elasticsearch 默认值，来自 RRF 论文）
- 不在某个检索器 top-N 里的文档，这个检索器对它贡献 0 分

#### 代码：`retrieval/rrf.py`

```python
"""RRF 融合算法"""
from collections import defaultdict


def rrf_fuse(rankings_list, k=60, topk=3):
    """
    rankings_list: list of list,每个子 list 是一个检索器的 top-N 结果,
                   每项是 (doc_id, score) 或只是 doc_id
    k: RRF 平滑常数
    topk: 最终返回前几条
    
    返回: [(doc_id, rrf_score), ...] 按 rrf_score 降序
    """
    rrf_scores = defaultdict(float)
    
    for ranking in rankings_list:
        for rank, item in enumerate(ranking, start=1):
            doc_id = item[0] if isinstance(item, (tuple, list)) else item
            rrf_scores[doc_id] += 1.0 / (k + rank)
    
    # 按 RRF 得分降序
    sorted_items = sorted(rrf_scores.items(), key=lambda x: x[1], reverse=True)
    return sorted_items[:topk]


# 简单测试
if __name__ == "__main__":
    bm25_results = [("doc_A", 10.5), ("doc_B", 9.3), ("doc_C", 8.1)]
    dense_results = [("doc_B", 0.92), ("doc_D", 0.88), ("doc_A", 0.85)]
    
    fused = rrf_fuse([bm25_results, dense_results], k=60, topk=3)
    print("RRF Top-3:")
    for doc_id, score in fused:
        print(f"  {doc_id}: {score:.4f}")
    
    # 预期: doc_A 和 doc_B 都在两个榜单里,分数最高
    # doc_A rank: 1(BM25) + 3(Dense) -> 1/61 + 1/63
    # doc_B rank: 2(BM25) + 1(Dense) -> 1/62 + 1/61
    # doc_B 应该略高于 doc_A
```

#### 为什么是 k=60

**k 的作用**：平滑 rank=1 和 rank=N 的差异。k 越大，排名前后的权重越均匀。

- k=0：完全依赖排名，第 1 名权重 1，第 2 名权重 0.5，差距悬殊
- k=60（RRF 论文和 Elasticsearch 默认）：第 1 名权重 1/61，第 2 名 1/62，差距小，更平滑
- k=∞：所有排名权重相同，退化为 "是否出现在榜单里"

k=60 是经过工业验证的甜点值，**除非有特殊原因，用这个值就好**。

### 五、FastAPI 检索服务

#### 要点

把上面三个部件封装成一个 HTTP 服务，提供 CLAUDE.md §4.1 规定的 `POST /retrieve` 接口。

#### 设计原则

1. **启动时加载所有索引到内存**（BM25 需要 JVM 启动约 10 秒，Dense 索引 85GB 但用 memmap 不占内存）
2. **单 worker 单线程**——训练进程串行调用；FAISS 和 Pyserini 都不是线程安全的
3. **passage 内容截断到 500 字符**（CLAUDE.md §3.1）——避免污染模型上下文
4. **各检索器各取 top-100**（CLAUDE.md §4.2），RRF 融合后返回 top-3
5. **日志详细**：每次检索记录 query、耗时、返回的 docids

#### 代码：`retrieval/server.py`

```python

"""

        logging.StreamHandler(),
        logging.FileHandler("logs/retrieval.log")

logger.info("加载 BM25 searcher...")
bm25_searcher = LuceneSearcher(BM25_INDEX_DIR)

logger.info("加载 FAISS 索引...")
faiss_index = faiss.read_index(f"{DENSE_INDEX_DIR}/faiss.index")

logger.info("加载 ID 映射...")
with open(f"{DENSE_INDEX_DIR}/ids.txt") as f:
    dense_ids = [line.strip() for line in f]
logger.info(f"Dense 索引有 {len(dense_ids)} 条 passage")

logger.info("加载 E5 模型...")
device = "cuda" if torch.cuda.is_available() else "cpu"
tokenizer = AutoTokenizer.from_pretrained(E5_MODEL_NAME)
model = AutoModel.from_pretrained(E5_MODEL_NAME).to(device).eval()
if device == "cuda":
    model = model.half()

logger.info("加载 passage 原文到内存(用于返回内容)...")
passage_store = {}
with open(CORPUS_PATH, encoding='utf-8') as f:
    for line in f:
        ex = json.loads(line)
        passage_store[ex["id"]] = ex["contents"]
logger.info(f"加载 {len(passage_store)} 条 passage 原文(约 10GB 内存)")

logger.info("✅ 所有资源加载完成,服务 Ready")


# ========== 检索函数 ==========
@torch.no_grad()
def encode_query(query: str) -> np.ndarray:
    """E5 要求 query 加 'query: ' 前缀"""
    enc = tokenizer([f"query: {query}"], padding=True, truncation=True,
                    max_length=128, return_tensors="pt").to(device)
    out = model(**enc)
    hidden = out.last_hidden_state.masked_fill(
        ~enc['attention_mask'][..., None].bool(), 0
    )
    emb = hidden.sum(1) / enc['attention_mask'].sum(1)[..., None]
    emb = torch.nn.functional.normalize(emb, p=2, dim=1)
    return emb.float().cpu().numpy()


def bm25_search(query: str, topn: int) -> list[tuple[str, float]]:
    hits = bm25_searcher.search(query, k=topn)
    return [(h.docid, h.score) for h in hits]


def dense_search(query: str, topn: int) -> list[tuple[str, float]]:
    emb = encode_query(query)
    D, I = faiss_index.search(emb, topn)
    return [(dense_ids[idx], float(score)) for idx, score in zip(I[0], D[0])]


def get_passage_content(doc_id: str) -> str:
    content = passage_store.get(doc_id, "")
    if len(content) > MAX_PASSAGE_CHARS:
        content = content[:MAX_PASSAGE_CHARS] + "..."
    return content


# ========== HTTP 接口 ==========
class RetrieveRequest(BaseModel):
    query: str
    topk: int = 3
    # 'fusion' 是消融实验扩展字段,CLAUDE.md spec 不要求,默认 rrf
    fusion: Literal["rrf", "bm25_only", "dense_only"] = "rrf"


@app.post("/retrieve")
def retrieve(req: RetrieveRequest) -> dict:
    """CLAUDE.md §4.1 接口:返回 {query, results: [{docid, score, contents}]}"""
    t0 = time.time()
    query = req.query.strip()
    if not query:
        raise HTTPException(status_code=400, detail="empty query")

    if req.fusion == "bm25_only":
        ranked = bm25_search(query, topn=req.topk)
    elif req.fusion == "dense_only":
        ranked = dense_search(query, topn=req.topk)
    else:
        bm25_r = bm25_search(query, topn=TOPN_PER_RETRIEVER)
        dense_r = dense_search(query, topn=TOPN_PER_RETRIEVER)
        ranked = rrf_fuse([bm25_r, dense_r], k=RRF_K, topk=req.topk)

    results = [
        {"docid": doc_id, "score": float(score), "contents": get_passage_content(doc_id)}
        for doc_id, score in ranked
    ]

    latency_ms = int((time.time() - t0) * 1000)
    logger.info(f"query={query[:60]!r} | topk={req.topk} | fusion={req.fusion} | latency={latency_ms}ms")

    # latency_ms / query 是对 CLAUDE.md spec 的扩展返回,不破坏契约
    return {"query": query, "results": results, "latency_ms": latency_ms}


@app.get("/health")
def health() -> dict:
    return {"status": "ok"}


if __name__ == "__main__":
    os.makedirs("logs", exist_ok=True)
    # 单 worker 保证 FAISS / Pyserini 的单线程访问(两者非线程安全)
    uvicorn.run(app, host="127.0.0.1", port=8000, log_level="info")
```

#### 坑点 8：内存占用

服务启动后内存占用约 **15-20GB**（与 CLAUDE.md §6.4 的 ~20GB 估计一致）：

- passage 原文 dict：10GB
- FAISS 索引：内存里只存指针结构，真正的 85GB 用 memmap 不占物理内存
- E5 模型（fp16）：250MB
- BM25（JVM）：2-3GB

AutoDL 3090 实例默认 30-60GB 内存，**完全够**。

#### 坑点 9：uvicorn 必须单 worker

默认 `uvicorn.run(...)` 就是单进程单 worker，不要加 `--workers N` 或 `workers=N`。Pyserini 的 `LuceneSearcher` 和 FAISS 都不是线程安全的，多 worker 会乱序返回或 crash。训练期间反正是串行调用，单 worker 最稳。

如果未来评测需要并发，可以考虑在 uvicorn 外面套一层进程级请求队列，但训练阶段用不到。

#### 坑点 10：E5 的 query 必须加 "query: " 前缀

**这是 E5 模型的硬要求**（论文和 Model Card 都明确说了）。不加前缀检索效果会严重下降：

- **passage 编码时**：加 `"passage: "` 前缀（在 `build_dense_index.py` 里）
- **query 编码时**：加 `"query: "` 前缀（在 `server.py` 里）

两边必须对应。

#### 坑点 11：如果要从 ModelScope 加载 E5

ModelScope 上有 E5-base 的镜像：`AI-ModelScope/e5-base-v2`。但**和 HF 版的参数完全相同**，用哪个都行。如果 HF 下载慢，改这里：

### 七、训练端调用契约

训练进程（`training/rollout_env.py`）通过 `requests.post` 调服务。CLAUDE.md §4.3 规定训练脚本必须在启动时做健康探活，失败则立刻 fail-fast：

```python
# training/grpo_train.py 启动前
import requests
try:
    r = requests.post(
        "http://127.0.0.1:8000/retrieve",
        json={"query": "test", "topk": 1},
        timeout=5,
    )
    assert r.status_code == 200
except Exception as exc:
    raise RuntimeError(
        "检索服务器未启动!先运行: bash scripts/start_retrieval.sh"
    ) from exc
```

**不要**把健康检查放在第一次 rollout 时才发现，否则 GRPO 已经 warmup 完 LoRA adapter、加载完 8×1024 的生成 batch，失败成本很高。
